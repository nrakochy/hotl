import { Card, CardGrid } from '@astrojs/starlight/components';

<CardGrid>
	<Card title="Nothing is ever lost" icon="document">
		Sessions are append-only event logs: `hotl resume` continues one,
		`hotl undo` reverses the agent's file changes via shadow-git snapshots,
		and context compaction adds a summary on top of history instead of
		rewriting it. [Overview →](./overview/)
	</Card>
	<Card title="Context that stays slim" icon="random">
		The window is spent on your task, not on ceremony:

		- Oversized tool results are evicted to files — a preview stays inline,
		  the agent pages the rest back on demand
		- Compaction summaries precompute in the background from ~60% full and
		  fold in at ~80% without pausing the session
		- The prompt prefix is byte-stable, so provider caches keep hitting
		- A per-turn meter shows how full the context actually is

		[Configuration →](./configuration/)
	</Card>
	<Card title="You own the loop" icon="pen">
		Steer mid-turn without killing the run — queued steers apply at the next
		step. And approvals are a vocabulary, not a y/N:

		- allow, or allow with an edited input
		- deny with a message the model actually reads
		- answer as the tool yourself
		- allow-rules absorb the repetitive asks, so approval fatigue never
		  trains you to say yes

		[Permissions & sandbox →](./permissions-and-sandbox/)
	</Card>
	<Card title="tmux-native, out of the box" icon="laptop">
		`hotl watch` in a pane discovers every AI agent in your tmux session:

		- live status per agent, a ping when one blocks on you, `enter` jumps
		  focus to its pane
		- `Ctrl-h/j/k/l` pane navigation composes with vim-tmux-navigator
		- vim hands welcome: the console's input editor speaks motions,
		  operators, and counts (`vim_mode`, on by default; `Ctrl-e` opens
		  `$EDITOR`), and watch's list speaks `j`/`k`/`gg`/`G`
		- the zsh plugin's OSC-133 marks give watch clean command boundaries

		[Shell integration →](./shell/)
	</Card>
	<Card title="It's Rust, and it goes brrrr" icon="seti:rust">
		One static binary — no Node, no Python, no runtime to install — and an
		async policy strict enough to keep it that way:

		- **Rule 0: async is for waiting, threads are for working.** The tokio
		  runtime only ever hosts tasks blocked on I/O or doing sub-millisecond
		  coordination
		- every entry point builds a `current_thread` runtime — nothing pays for
		  thread-pool spinup on the cold-start path that `hotl -p` and the zsh
		  plugin live on
		- shadow-git snapshots and other hot work leave the runtime via
		  `spawn_blocking` rather than stalling the event loop
		- four `unsafe` blocks in the entire workspace, every one a plain libc
		  syscall for the sandbox or a process-group kill

		[Architecture →](./overview/)
	</Card>
	<Card title="Concurrency you can reason about" icon="rocket">
		One actor per session is the sole committer, so there is exactly one
		writer and no lock order to get wrong. Around that spine:

		- tool batches chunk themselves — contiguous runs of parallel-safe calls
		  (reads, isolated sub-agents) overlap under one `join_all`; anything
		  that mutates runs alone, in source order, bracketed by undo snapshots
		- results come back paired to their call either way, so parallelism
		  never reorders the transcript
		- approvals stay deliberately serial: an ask is a human moment, not a
		  race
		- `Ctrl-c` never queues — it trips a cancellation token out-of-band, so
		  interrupts can't wedge behind a full mailbox
		- compaction is speculative: the digest fires as a background task at
		  ~60% context and is usually already done when the fold lands

		[Overview →](./overview/)
	</Card>
	<Card title="Any model, one env var" icon="setting">
		`HOTL_MODEL=provider/model` — Anthropic's Messages API, or any
		OpenAI-compatible endpoint: OpenAI, Groq, a local Ollama server, or a
		gateway like LiteLLM. [Gateways & key sources →](./gateway/)
	</Card>
	<Card title="Standard protocols at the edges" icon="puzzle">
		An MCP client brings in external tools; `hotl acp` embeds the agent in
		any ACP-speaking editor — and your Claude Code skills
		(`~/.claude/skills`, plugins) load in place, no porting.
		[MCP servers →](./mcp/)
	</Card>
	<Card title="One config file, fail-closed" icon="document">
		Everything hand-editable lives in a single `config.toml` — no
		`permissions.toml`/`mcp.toml`/`hooks.toml` sprawl:

		- precedence is always env var > config.toml > default, so CI can
		  override anything without editing the file
		- a malformed file is ignored with a warning, never half-applied; a
		  typo'd permission mode fails closed to `ask`
		- API keys never live in the file — env vars, or `api_key_helper`
		  runs a command and uses its output
		- `hotl setup` writes a commented starter; `hotl doctor` checks the lot

		[Configuration →](./configuration/)
	</Card>
	<Card title="Security floor that never turns off" icon="approve-check">
		Every mutating tool call passes one fixed gate, and what sits under it has
		no off switch — in any mode:

		- `bash` runs confined by the kernel: Seatbelt on macOS, Landlock on
		  Linux ≥ 5.13 including WSL2. Writes land in the working directory and
		  temp, nowhere else
		- where the floor can't be enforced, hotl degrades **fail-closed** —
		  every exec is individually human-gated with an `UNSANDBOXED` banner and
		  bash allow-rules stop applying
		- protected paths (git hooks, shell rc, Makefiles, `.ssh/`, credential
		  stores, agent-instruction files) are checked *before* allow rules and
		  never auto-approve
		- optional egress restriction is kernel-backed, not proxy-backed: a
		  client that ignores the proxy hits a loopback-only wall
		- secrets are masked at log ingestion, and secret-bearing files never
		  enter the undo snapshot store at all
		- *forthcoming:* third-party plugins as WASM components under a
		  wasmtime sandbox (epoch deadlines + fuel) — that lane doesn't ship
		  until its sandbox and trust screens do

		Stated honestly: this is write-confinement, not exfiltration prevention.
		[Permissions & sandbox →](./permissions-and-sandbox/)
	</Card>
</CardGrid>
