# Starlight docs site, redeployed on each release

Date: 2026-07-21
Status: approved

## Goal

Publish hotl's user docs (`docs/user/*.md`) as a Starlight site. The site redeploys
automatically whenever a hotl release is published, so the live docs always match the
latest released version. Host on GitHub Pages now; migrate to Cloudflare Pages with a
custom domain later.

## Decisions

- **Update trigger:** workflow fires on `release: published` (the event cargo-dist's
  autogenerated `release.yml` produces when it creates the GitHub Release), plus
  `workflow_dispatch` for manual redeploys. This means docs deploy only for real,
  successful releases, and no hand edits to the cargo-dist-owned `release.yml`
  (which `dist init` regenerates) are needed.
- **Canonical docs source:** files move from `docs/user/` into
  `site/src/content/docs/`. `docs/user/` is deleted; in-repo links updated.
- **Hosting:** GitHub Pages (Source: GitHub Actions) at `nrakochy.github.io/hotl`.
  Custom domain recommended early so links survive the later Cloudflare move.
- **Host-specific config isolation:** all host-specific bits live in
  `site/astro.config.mjs` (`site`, `base`) and `.github/workflows/docs.yml`, so the
  Cloudflare migration touches only those two files.

## Components

- `site/` — Astro + Starlight app, dependencies pinned via `package-lock.json`.
  - `astro.config.mjs`: `site: 'https://nrakochy.github.io'`, `base: '/hotl'`,
    explicit sidebar in reading order: Quickstart → Configuration → Permissions →
    TUI → Backgrounding → Hooks → MCP → Gateway → Troubleshooting → Uninstall.
  - `src/content/docs/` — the ten migrated pages; each gains required `title:`
    frontmatter (from its former H1); `index.md` is the landing page.
- `.github/workflows/docs.yml` — on `release: published` / `workflow_dispatch`:
  checkout the released tag (`github.event.release.tag_name`) so content matches the
  release, `npm ci && npm run build` in `site/`, deploy via
  `actions/upload-pages-artifact` + `actions/deploy-pages`.
- `ci.yml` guard job — builds the site on PRs/pushes touching `site/**` so breakage
  surfaces before release day.

## Manual steps (owner)

- Enable Pages in repo settings: Source = GitHub Actions (one-time).
- Optionally attach a custom domain before publicizing links.

## Testing

`npm run build` is the verification (Starlight fails the build on broken internal
links). Run locally during migration; enforced continuously by the CI guard job.

## Cloudflare migration (later)

Create a Cloudflare Pages project for the repo (or deploy via wrangler from the same
workflow), set the custom domain, update `site`/drop `base` in `astro.config.mjs`,
disable GitHub Pages.
